Theme — light base, royal override

The system ships a light base (cream page, white cards, ink text). tokens/theme-royal.css is imported last and flips the semantic surface/text aliases to the royal-green dark theme — which is what the whole site renders. Author against semantic tokens (--bg-canvas, --text-body) and royal comes for free.

Shadows use warm ink rgba(20,22,13,…), never pure black. Motion is fast & springy.

⚠️ prefers-reduced-motion: reduce is honoured globally (base.css) — animations collapse to ~0.01ms. Keep transforms-only hovers (150–300ms); never instant 0ms state changes (MASTER anti-pattern).

Anti-patterns

Hard "no" list — enforced on every PR (full checklist in design-system/MASTER.md).

  • ❌ Emojis as icons → use the SVG set in components/Icons.jsx
  • ❌ outline:none without the global focus shadow
  • ❌ Color-only meaning (always pair an icon/text — diets, allergens, macros)
  • ❌ Off-brand fonts (Playfair/Karla) or raw hex in components — use tokens
  • ❌ Layout-shifting hovers; instant (0ms) state changes
  • ❌ --text-on-dark-faint on small text (fails AA below 24px)
